Goal: 100% reliable CAN TX callback path on FDCAN. Eliminate pollTxCallback workaround.

Scope

  • Verify: TC bit clears correctly, IE.TCE enables at right time

PR B — STM32 Platform Cleanup

Priority: High  |  Fixes: DEV-01, LD-01, LD-03, CLK-03, F-01

Goal: Harden platform for production use — stack safety, encapsulation, config hygiene.

Scope

  • Make 6 FdCanDevice internal methods private
  • Increase _Min_Stack_Size from 0x400 → 0x1000 (4 KB)
  • Add linker ASSERT((_end + _Min_Heap_Size) < _estack - _Min_Stack_Size)
  • Add F4 flash latency readback verification loop
  • Either set configUSE_MALLOC_FAILED_HOOK = 1 or remove dead hook function
  • Verify no build regressions on all 4 presets (FreeRTOS + ThreadX x 2 boards)

PR C — STM32 DMA UART

Priority: Medium  |  New feature

Goal: Non-blocking UART for production console. Current polling UART blocks ~7ms per print in CAN task context.

Scope

  • DMA TX channel for USART3 (F4) / LPUART1 (G4)
  • Ring buffer RX with idle-line detection (IDLEIE)
  • DMA transfer-complete ISR with callback
  • Unit tests for DMA TX/RX paths
  • HIL verify: console at 115200 under CAN + UDS load

PR E — STM32 Ethernet (G474RE)

Priority: Low  |  Full stack parity with S32K

Goal: Bring Ethernet/DoIP to STM32. Note: G474RE has no built-in Ethernet MAC — would need SPI Ethernet module (e.g., W5500) or target a different STM32 with MAC (e.g., STM32F4x7/F4x9).

Scope

  • Evaluate: SPI-based W5500 vs. STM32F4x7 native MAC
  • Register-level MAC driver (no HAL)
  • lwIP integration matching POSIX/S32K pattern
  • UDP echo server demo
  • HIL: ping + UDP throughput under CAN load
