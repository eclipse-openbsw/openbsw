HIL Open Items

FDCAN TC ISR Delivery (~14%)

Symptom: TC (Transmission Complete) interrupt fires for only ~7-14% of TX completions on G474RE FDCAN.

Workaround: pollTxCallback every 50ms in TASK_CAN context handles the remaining completions.

Impact: Fire-and-forget CAN TX works 100%. UDS listener-based TX (which requires async callback) only works ~14%.

Investigation status:

  • ISR priority fixed: 5→6 (FreeRTOS safe)
  • Clear stale TC before enabling TCE
  • S32K-matching code structure verified (1622 unit tests pass)
  • Root cause: FDCAN TC interrupt timing vs. FreeRTOS scheduler — needs scope analysis
DoCAN Multi-SF (Upstream Issue)

Symptom: handlePendingMessageReceivers blocks consecutive Single Frame requests on the same CAN ID. ~10% delivery rate.

Owner: Upstream (eclipse-openbsw/openbsw). Not our code.

Impact: Rapid UDS request sequences fail. Single requests work fine.

  • Needs investigation in libs/bsw/docan/
  • Likely dedup logic in asyncImpl shared across platforms
